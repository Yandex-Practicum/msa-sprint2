# Миграция монолита Hotelio на микросервисы

## Что сделано

В рамках перехода на микросервисную архитектуру созданы два новых сервиса:

### 1. Booking Service (микросервис бронирований)

- Реализован на Node.js с использованием gRPC
- Имеет собственную базу данных `booking_db` (PostgreSQL)
- Принимает запросы на создание бронирований от монолита через gRPC
- Отправляет события о новых бронированиях в Kafka

### 2. Booking History Service (микросервис истории)

- Реализован на Node.js
- Имеет собственную базу данных `history_db` (PostgreSQL)
- Слушает события из Kafka (топик `BookingCreated`)
- Сохраняет исторические данные о бронированиях для аналитики

## Как работает взаимодействие

1. Монолит получает REST запрос на создание бронирования
2. Монолит вызывает gRPC метод `CreateBooking` в `booking-service`
3. `booking-service` сохраняет бронирование в своей БД
4. `booking-service` отправляет событие в Kafka
5. `booking-history-service` получает событие и сохраняет историю

## Целевая стратегия (To Be)

После завершения миграции всех сервисов планируется:

### 1. Перенос исторических данных
- Написать скрипт для переноса старых бронирований из монолита в `booking_db` и `history_db`
- Использовать Kafka для асинхронного переноса (чтобы не нагружать систему)

### 2. Полное отключение старой логики
- Удалить код бронирований из монолита
- Переключить `GET /api/bookings` на вызов `booking-service` через gRPC
- Удалить таблицу `booking` из монолитной БД

## Результат

- Создание бронирований вынесено в отдельный микросервис
- Исторические данные собираются асинхронно без нагрузки на основную БД
