### Объяснение стратегии миграции данных при запуске нового сервиса и стратегии To Be

- монолит перенаправляет запрос на создание бронирования через gRPC на новый микросервис бронирования
- в микросервисе бронирования при создании бронирования вызываются REST endpoints монолита для получения валидируемых полей и при успешном прохождении валидации бронирование записывается в БД микросервиса
