### Этапы миграции
1. Перенос данных
    - Экспорт данных из монолита.
    - Импорт в новую базу.

2. Временная синхронизация данных:
    - Монолит пишет в старую БД и дублирует в gRPC.
    - Новый сервис публикует события в Kafka для аналитика.

3. Очистка:
    - После проверки работоспособности и оптимизации - отключение записи в старую БД

### Целевая архитектура
- Микросервисы: Booking, User, Hotel, Promo, Review — через Kafka.
- Взаимодействие: gRPC для синхронного, Kafka - для событий и асинхронных данных.
- Данные: отдельная БД для каждого сервиса.
- Деплоймент: В Kubernetes с использованием Istio для маршрутизации.
- Фронтенд: Запросы через GraphQL federation.
