# Управление трафиком с Istio

## Что сделано

### 1. Установлен Istio в Minikube

### 2. Развёрнуты две версии сервиса
- **v1** — основная версия, фича-флаг выключен (`ENABLE_FEATURE_X=false`)
- **v2** — новая версия, фича-флаг включён (`ENABLE_FEATURE_X=true`)

### 3. Настроена канарейка
- 90% трафика идёт на v1
- 10% трафика идёт на v2
- Настройка через VirtualService с `weight`

### 4. Настроен фича-флаг через заголовок X-Feature-Enabled
- Запросы с этим заголовком идут на v2

### 5. Настроен Retry и Circuit Breaker
- При ошибке запрос повторяется до 3 раз
- При повторяющихся ошибках сервис временно отключается (circuit breaker)

### 6. Настроен Fallback-маршрут
- При недоступности v1 трафик переключается на v2

## Вывод

Скрипты проверены. Задание выполнено
