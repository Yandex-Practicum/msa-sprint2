### Внесённые изменения

- Сервис `booking-service` взят из задания 4 и адаптирован под управление через Helm.
- Добавлен новый Helm-чарт с шаблонами для Deployment, Service и Istio-ресурсов.
- Поддержка версий `v1` и `v2` реализована через параметры Helm (`--set version=v1`, `--set version=v2`); конфигурация фича-флага (`ENABLE_FEATURE_X`) управляется через переменные в `values.yaml`.
- Установлен Istio (профиль `demo`) в Minikube; включена автоматическая инъекция sidecar в namespace `default`.
- Конфигурация Istio (`VirtualService`, `DestinationRule`) вынесена в отдельные манифесты и применяется вручную (вне Helm), чтобы избежать конфликтов при управлении глобальными ресурсами.
- Настроена маршрутизация:
  - Канареечный релиз: 90% трафика → `v1`, 10% → `v2`,
  - Фича-флаг: запросы с заголовком `X-Feature-Enabled: true` направляются на `v2`,
  - Настроены retry и circuit breaking через `DestinationRule`.
- Исправлен CI/CD: обновлены шаги деплоя для развёртывания двух версий сервиса.
- Основные команды (сборка, деплой, проверка) вынесены в `Makefile` для удобства запуска.
