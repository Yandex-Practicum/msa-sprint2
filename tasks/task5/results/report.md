# Задание 5: Настройка управления трафиком с Istio

## Описание изменений и решений

### Цель задания
Настроить маршрутизацию трафика с Istio для сервиса booking-service с канареечным развертыванием, fallback маршрутизацией и feature flags.

### Выполненные задачи

#### 1. Установка Istio
- Установлен Istio в Minikube с профилем demo
- Включена автоматическая инъекция Istio в default namespace
- Проверена корректность установки

#### 2. Развертывание двух версий сервиса
- **v1** - основная версия (2 реплики)
- **v2** - новая версия с feature flags (1 реплика)
- Обе версии работают с Istio sidecar

#### 3. Настройка Istio маршрутизации

##### VirtualService
- Канареечное развертывание: 90% трафика на v1, 10% на v2
- Feature flag маршрутизация: заголовок X-Feature-Enabled: true направляет на v2
- Настроены retries (3 попытки) и timeout (5 секунд)

##### DestinationRule
- Настроен Circuit Breaking с outlier detection
- Connection pooling для оптимизации производительности
- Различные настройки для v1 и v2

##### EnvoyFilter
- Добавление заголовка x-feature-version для v2
- Корректная работа с feature flags

#### 4. Тестирование функциональности

##### Канареечное развертывание
- Проверено распределение трафика между версиями
- 20 тестовых запросов выполнены успешно

##### Feature Flag маршрутизация
- Заголовок X-Feature-Enabled: true корректно направляет на v2
- Обычные запросы направляются согласно канареечному развертыванию

##### Fallback маршрутизация
- При ошибках v1 трафик перенаправляется на v2
- Система устойчива к отказам отдельных версий

### Технические детали

#### Архитектура
- Istio Service Mesh с автоматической инъекцией
- Два Deployment с разными версиями
- Единый Service для внешнего доступа
- Istio ресурсы для управления трафиком

#### Безопасность
- Таймауты на все запросы (5 секунд)
- Circuit breaking для защиты от каскадных отказов
- Retry логика с ограничениями

#### Мониторинг
- Логи Istio sidecar доступны
- Метрики Envoy proxy включены
- Health checks для всех версий

### Результат
Система успешно настроена с Istio Service Mesh. Канареечное развертывание работает корректно, feature flags функционируют, fallback маршрутизация обеспечивает отказоустойчивость. Все проверочные скрипты выполняются успешно.
